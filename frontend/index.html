⬡ Core Linux APIs Used
  • fork() — Process creation
  • exec() — Program execution
  • wait() / waitpid() — Reap children
  • kill() — Signal delivery
  • nice() / setpriority() — Scheduling
  • getpid() / getppid() — PID queries
  • sigaction() — Signal handlers
  • /proc filesystem — Kernel data
⎇ Process Lifecycle
  • 1. New — fork() called
  • 2. Ready — In scheduler queue
  • 3. Running — On CPU
  • 4. Waiting — I/O blocked
  • 5. Stopped — SIGSTOP received
  • 6. Zombie — Exited, not reaped
  • 7. Dead — waitpid() completed
◈ Signal Reference
  • SIGTERM (15) — Graceful shutdown
  • SIGKILL (9) — Forced kill
  • SIGSTOP (19) — Pause execution
  • SIGCONT (18) — Resume process
  • SIGHUP (1) — Reload config
  • SIGCHLD (17) — Child state change
  • SIGINT (2) — Ctrl+C interrupt
